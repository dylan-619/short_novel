# 读者反馈

## 【执行模式】单章节处理

**【关键】本技能必须**：
- 每次只处理一个章节
- 等待反馈完成后才返回结果
- 不得批量处理多个章节
- 不得跳过读者反馈

## 职责

模拟番茄平台读者视角评价章节和全文，提供阅读体验反馈。

## 输入格式

### 章节反馈

```json
{
  "feedback_type": "chapter",
  "chapter_number": 1,
  "chapter_content": "章节内容...",
  "target_audience": "female",
  "target_tone": "虐恋爽感",
  "work_dir": "./outputs/[书名]/work/"
}
```

**【关键】输出路径规则（所有技能必须统一）**：
- 工作目录：`./outputs/[书名]/work/`
- 参考资料：`./docs/参考资料.md`
- 输出目录：`./outputs/`
- 小说目录：`./outputs/[书名]/`
- 临时文件必须输出到 `work_dir`，禁止输出到 `outputs/` 根目录

### 全文反馈

```json
{
  "feedback_type": "full",
  "full_content": "全文内容...",
  "target_audience": "female",
  "target_tone": "虐恋爽感",
  "meta": {
    "total_chapters": 20,
    "total_words": 18000
  }
}
```

## 评估维度

### 1. 开篇吸引力

**检查项**：
| 检查项 | 权重 | 评估标准 |
|--------|------|----------|
| 300字内出冲突 | 0.4 | 必须满足 |
| 冲突强烈度 | 0.3 | 明显/强烈 |
| 人物出场清晰 | 0.2 | 主角反派明确 |
| 开篇钩子强 | 0.1 | 有悬念/期待 |

**评分标准**：
| 分数 | 等级 | 说明 |
|------|------|------|
| 9.0+ | 优秀 | 开篇完美，冲突强烈 |
| 8.0-8.9 | 良好 | 开篇清晰，冲突明显 |
| 7.0-7.9 | 一般 | 开篇尚可，冲突一般 |
| <7.0 | 较差 | 开篇平淡，冲突不足 |

## 反馈格式

### 亮点描述

```
"开篇冲突强烈，300字内出高潮"
"女主性格鲜明，心死觉醒点清晰"
"陆沉渣男形象立得住，令人反感"
"章末钩子强，有强烈期待感"
```

### 不足描述

```
"部分对话略显生硬"
"心理描写可以再细致一些"
"第8章节奏稍慢"
"部分章节爽点不够密集"
```

### 建议描述

```
"建议在第 3 段增加女主心理细节"
"对话可以更自然一些"
"第8章建议加快节奏"
"第12-13章爽点可以更密集"
```

## 注意事项

1. 评估需客观，避免主观偏见
2. 亮点和不足并重
3. 建议具体可执行
4. 评分需符合平台标准
5. 考虑读者群体特征
6. 评估维度全面
7. 输出格式统一
