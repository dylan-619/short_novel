# 基调维持

## 【执行模式】单次检查

**【关键】本技能必须**：
- 每次只检查一个全文
- 等待检查完成后才返回结果
- 不得跳过章节级检查
- 必须检查所有章节

## 职责

确保全文风格统一，基调一致，阅读体验流畅。

## 输入格式

```json
{
  "full_draft": {
    "chapters": [
      {
        "chapter_number": 1,
        "content": "章节内容...",
        "word_count": 856
      },
      ...
    ]
  },
  "target_tone": "虐恋爽感",
  "session_id": "uuid-v4",
  "creation_timestamp": "2026-03-20T10:30:00Z",
  "work_dir": "./outputs/[书名]/work/",
  "reference_path": "./docs/参考资料.md"
}
```

**【关键】输出路径规则（所有技能必须统一）**：
- 工作目录：`./outputs/[书名]/work/`
- 参考资料：`./docs/参考资料.md`
- 输出目录：`./outputs/`
- 小说目录：`./outputs/[书名]/`
- 临时文件必须输出到 `work_dir`，禁止输出到 `outputs/` 根目录
```

## 输出格式

```json
{
  "consistency_report": {
    "overall_score": 9.5,
    "tone_type": "虐恋爽感",
    "consistency_level": "high",
    "deviations": [
      {
        "chapter": 8,
        "location": "paragraphs 12-14",
        "issue": "风格偏甜宠，与虐恋基调不符",
        "suggestion": "建议调弱甜腻感，保持虐恋氛围"
      }
    ],
    "standardized_text": {
      "chapters": [
        {
          "chapter_number": 1,
          "content": "统一后内容...（或未修改则标记为 null）"
        },
        ...
      ]
    }
  }
}
```

## 基调类型

### 1. 虐恋爽感

**特征**：
- 前期虐（女主受伤害）
- 中期觉醒（女主反击）
- 后期爽（打脸复仇）

**风格要求**：
| 元素 | 要求 |
|------|------|
| 语言风格 | 冷峻/犀利 |
| 对话 | 有力、不拖沓 |
| 心理描写 | 突出痛苦/觉醒/爽快 |
| 节奏 | 紧凑、爽点密集 |

**典型场景**：
- 前期：冷漠、轻视、伤害
- 中期：冷淡、冷静、反击
- 后期：打脸、爽快、独白美

---

### 2. 甜宠治愈

**特征**：
- 全程高甜
- 温馨治愈
- 甜度拉满

**风格要求**：
| 元素 | 要求 |
|------|------|
| 语言风格 | 温柔/甜蜜 |
| 对话 | 宠溺、甜蜜 |
| 心理描写 | 突出幸福/心动 |
| 节奏 | 舒缓、甜蜜点多 |

---

### 3. 悬疑惊悚

**特征**：
- 紧张恐怖
- 悬念密集
- 反转不断

**风格要求**：
| 元素 | 要求 |
|------|------|
| 语言风格 | 紧张/压抑 |
| 对话 | 简洁、暗示性强 |
| 心理描写 | 突出恐惧/紧张 |
| 节奏 | 极快、悬念钩子强 |

---

### 4. 搞笑脑洞

**特征**：
- 轻松搞笑
- 脑洞清奇
- 反套路

**风格要求**：
| 元素 | 要求 |
|------|------|
| 语言风格 | 幽默/轻松 |
| 对话 | 诙谐、吐槽 |
| 心理描写 | 突出无奈/搞笑 |
| 节奏 | 轻快、笑点密集 |

## 一致性检查

### 检查维度

| 检查项 | 检测方法 |
|--------|----------|
| 语言风格 | 统计形容词、句式使用 |
| 对话风格 | 检测对话语气、用词 |
| 心理描写 | 检测心理描写频率、风格 |
| 节奏风格 | 检测段落长度、情节密度 |
| 情绪基调 | 检测情绪词分布 |

### 偏差检测

**检测流程**：
1. 建立基调模型（前3章作为基准）
2. 分析后续章节
3. 识别风格偏离
4. 计算偏差分数

**偏差等级**：
| 等级 | 偏差分数 | 处理方式 |
|------|----------|----------|
| 无偏差 | 0-2 | 无需调整 |
| 轻微偏差 | 2-5 | 建议微调 |
| 明显偏差 | 5-8 | 需要调整 |
| 严重偏差 | >8 | 必须修正 |

## 统一处理

### 修正方法

1. **语言风格统一**
   - 替换不符合基调的形容词
   - 调整句式风格
   - 统一修辞手法

2. **对话风格统一**
   - 调整对话语气
   - 精简或扩充对话
   - 统一称呼和用词

3. **心理描写统一**
   - 调整心理描写密度
   - 统一情绪表达方式
   - 确保符合人物设定

4. **节奏风格统一**
   - 调整段落长度
   - 平衡情节密度
   - 统一叙述节奏

### 示例修正

**虐恋爽感修正**：
```json
{
  "chapter": 8,
  "original": "\"亲爱的，你终于醒了，我好担心你。\"陆沉深情地看着林婉。",
  "issue": "过于甜腻，与虐恋基调不符",
  "suggested": "陆沉冷眼看着林婉：\"醒了就起来，别装死。\""
}
```

**悬疑惊悚修正**：
```json
{
  "chapter": 12,
  "original": "房间里很安静，只有时钟滴答滴答的声音。",
  "issue": "悬疑感不足，过于平淡",
  "suggested": "房间里死一般寂静，只有时钟的滴答声像倒计时一样，一步步逼近..."
}
```

## 评分标准

```json
{
  "score_breakdown": {
    "language_consistency": 9.5,    // 语言风格一致性
    "dialogue_consistency": 9.2,    // 对话风格一致性
    "psychology_consistency": 9.0,   // 心理描写一致性
    "rhythm_consistency": 8.8,      // 节奏风格一致性
    "overall_score": 9.1             // 综合评分
  }
}
```

| 分数 | 一致性等级 | 说明 |
|------|-----------|------|
| 9.0+ | 优秀 | 基调高度统一 |
| 8.0-8.9 | 良好 | 基调基本统一 |
| 7.0-7.9 | 一般 | 存在明显偏差 |
|建议修正 | <7.0 | 较差 | 风格混乱，必须修正 |

##` 注意事项

1. 修正时不改变核心情节
2. 保持人物性格一致
3. 优先保留精彩片段
4. 记录所有修改便于审核
5. 评分需客观准确
