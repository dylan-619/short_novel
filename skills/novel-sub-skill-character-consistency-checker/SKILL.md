# 人物一致性检查

## 职责

确保人物行为、情感状态、关系进展在全文中逻辑连贯，避免人物状态前后矛盾。

## 检查维度

### 1. 行为一致性检查

**检查项**：
| 检查项 | 权重 | 说明 |
|--------|------|------|
| 行为是否符合弧光阶段 | 0.4 | 当前处于 start/middle/end 哪个阶段 |
| 行为是否符合人物设定 | 0.3 | 性格特点是否保持一致 |
| 行为是否符合情境 | 0.2 | 在特定情境下行为是否合理 |
| 行为是否有动机支撑 | 0.1 | 行为背后是否有合理动机 |

**弧光阶段行为规范**：

**女频追妻火葬场 - 女主**：
| 阶段 | 典型行为 | 禁忌行为 |
|------|----------|----------|
| start（深情付出） | 默默等待、委曲求全、关心对方 | 冷漠、反击、决绝 |
| middle（心死觉醒） | 冷静、筹划、反击、不再期待 | 深情付出、原谅太快、软弱 |
| end（独立女王） | 果断、强势、事业为重、打脸 | 犹豫、心软、回头 |

**女频追妻火葬场 - 男主**：
| 阶段 | 典型行为 | 禁忌行为 |
|------|----------|----------|
| start（冷漠渣男） | 轻视、不耐烦、偏爱白月光 | 后悔、关心女主、觉醒 |
| middle（幡然醒悟） | 焦虑、追妻、卑微、后悔 | 仍冷漠、不作为 |
| end（后悔追妻） | 跪舔、疯狂弥补、崩溃 | 仍傲慢、不承认错误 |

**男频赘婿逆袭 - 男主**：
| 阶段 | 典型行为 | 禁忌行为 |
|------|----------|----------|
| start（隐忍扮猪） | 装穷、受辱、隐忍不发 | 摊牌、反击、暴露实力 |
| middle（摊牌打脸） | 暴露身份、快速打脸、展现实力 | 继续隐忍、犹豫不决 |
| end（巅峰无敌） | 强势、掌控全局、所有人跪舔 | 软弱、妥协、被欺负 |

### 2. 关系递进检查

**检查项**：
| 检查项 | 权重 | 说明 |
|--------|------|------|
| 关系状态是否明确 | 0.3 | 当前关系是否清晰 |
| 关系变化是否有触发事件 | 0.3 | 关系变化是否合理 |
| 关系变化是否符合节奏 | 0.2 | 变化速度是否合理 |
| 关系变化是否有情感铺垫 | 0.2 | 是否有足够的铺垫 |

**关系状态分类**：
```
陌生 → 认识 → 熟悉 → 友好 → 亲密
  ↓
敌对 → 仇视 → 冷漠 → 缓和 → 和解
```

**关系强度等级**（1-10）：
| 强度 | 说明 | 示例 |
|------|------|------|
| 1-3 | 弱关系 | 路人、点头之交 |
| 4-6 | 中等关系 | 同事、普通朋友 |
| 7-8 | 强关系 | 好友、恋人、家人 |
| 9-10 | 极强关系 | 生死之交、深仇大恨 |

**关系变化合理性检查**：
```json
{
  "change_check": {
    "from_state": "深爱",
    "to_state": "心死",
    "trigger_event": "目睹出轨",
    "is_reasonable": true,
    "reason": "重大背叛事件足以导致感情破裂"
  }
}
```

### 3. 情感连贯性检查

**检查项**：
| 检查项 | 权重 | 说明 |
|--------|------|------|
| 情感状态是否与前章连贯 | 0.4 | 不能突然跳变 |
| 情感变化是否有原因 | 0.3 | 每次变化都要有触发 |
| 情感表现是否一致 | 0.2 | 表现方式是否符合设定 |
| 情感强度是否合理 | 0.1 | 强度是否符合情境 |

**情感状态分类**：
```
积极情感：期待、喜悦、满足、平静
消极情感：痛苦、愤怒、失望、绝望
中性情感：冷静、冷漠、麻木
```

**情感轨迹记录**：
```json
{
  "emotional_trajectory": {
    "chapter_1": "期待 → 震惊 → 痛苦 → 心死",
    "chapter_2": "麻木 → 冷静 → 筹备",
    "chapter_3": "坚定 → 反击开始"
  }
}
```

**情感连贯性检查示例**：
```json
{
  "check": {
    "previous_emotion": "心死",
    "current_emotion": "期待",
    "has_transition": false,
    "is_continuous": false,
    "issue": "情感突然从'心死'跳变到'期待'，缺乏过渡"
  }
}
```

### 4. 记忆一致性检查

**检查项**：
| 检查项 | 权重 | 说明 |
|--------|------|------|
| 是否记得已见过的人 | 0.3 | 不能重复"初次见面" |
| 是否记得发生的事件 | 0.3 | 不能忘记重要事件 |
| 是否记得说过的话 | 0.2 | 不能前后矛盾 |
| 是否记得人物关系 | 0.2 | 关系认知要一致 |

**常见记忆错误类型**：

| 错误类型 | 示例 | 修正建议 |
|----------|------|----------|
| 重复初见 | 第3章已见，第5章又写"初次见面" | 改为"又见面了" |
| 忘记事件 | 已知出轨，又表现出信任 | 保持怀疑态度 |
| 对话矛盾 | 之前说恨，现在又表现出爱 | 需要过渡情节 |
| 关系错乱 | 把敌人当朋友 | 保持敌对态度 |

---

### 5. 对话适当性检查

**检查项**：
| 检查项 | 权重 | 说明 |
|--------|------|------|
| 对话语气是否符合人物状态 | 0.4 | 冷漠/热情/敌意 |
| 对话内容是否符合关系 | 0.3 | 亲密/疏离/敌对 |
| 对话是否符合情境 | 0.2 | 公开/私下/冲突中 |
| 对话是否符合性格 | 0.1 | 果断/犹豫/直接 |

**对话语气与人物状态匹配**：

| 人物状态 | 对话特点 | 示例 |
|----------|----------|------|
| 冷漠 | 简短、无感情 | "好。"/"知道了。" |
| 愤怒 | 冲动、尖锐 | "你凭什么！" |
| 心死 | 平静、无所谓 | "随便你。" |
| 觉醒 | 冷静、有力量 | "我不会再妥协了。" |
| 反击 | 讽刺、有力 | "你以为我还会在乎吗？" |

## 检查流程

### 章节级检查（每章写完后）

1. **提取本章人物行为**
   - 识别主要人物行为
   - 分析行为动机
   - 判断行为类型

2. **对比弧光阶段**
   - 确定当前章节属于哪个阶段
   - 检查行为是否符合阶段规范
   - 识别不符合的行为

3. **检查关系变化**
   - 提取本章关系变化
   - 检查变化是否有触发事件
   - 评估变化合理性

4. **验证情感连贯**
   - 提取本章情感状态
   - 对比前章情感
   - 检查是否有过渡

5. **检查记忆一致**
   - 识别人物互动
   - 检查是否与历史记录冲突
   - 验证人物认知一致性

6. **生成检查报告**
   - 汇总所有问题
   - 给出修正建议
   - 更新人物状态

## 评分标准

| 分数 | 状态 | 说明 |
|------|------|------|
| 9.0+ | pass | 人物状态高度一致，无矛盾 |
| 8.0-8.9 | pass | 人物状态基本一致，个别微调 |
| 7.0-7.9 | needs_revision | 存在明显不一致，需要修改 |
| <7.0 | reject | 人物状态严重矛盾，需要重写 |

## 状态更新规则

### 更新触发条件

1. **关系状态变化**
   - 检测到明显的关系变化事件
   - 关系强度变化 >= 2
   - 关系性质改变（友好→敌对）

2. **情感状态变化**
   - 检测到明显的情感转折
   - 情感性质改变（积极→消极）
   - 情感强度显著变化

3. **重要事件影响**
   - 识别关键剧情事件
   - 分析事件对人物的影响
   - 记录影响程度和性质

### 更新格式

```json
{
  "character_state_update": {
    "江念": {
      "relationships": {
        "顾辞": {
          "history_addition": {
            "chapter": 5,
            "from": "仇视",
            "to": "彻底死心",
            "trigger": "发现顾辞仍在维护沈离",
            "intensity_change": 8 → 9
          }
        }
      },
      "emotional_trajectory_addition": {
        "chapter_5": "冷静 → 讽刺 → 坚定反击"
      },
      "key_events_impact_addition": [
```

## 注意事项

1. **严格按弧光阶段检查行为**
2. **关系变化必须有触发事件**
3. **情感变化必须有过渡**
4. **记忆要全程保持一致**
5. **对话要符合当前状态**
6. **输出修正建议必须具体可执行**
7. **状态更新要准确记录**
8. **评分要客观公正**
