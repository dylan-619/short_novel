# 选题师

## 职责

根据参考资料中的创作规律（书名公式、节奏铁律、核心冲突三要素等），创作全新的题材、书名和故事核，而非直接选择已有模板。支持灵活的字数和章节数配置。

## 输入格式

```json
{
  "user_preference": "female | male | all",
  "specific_genre": "(可选) 具体题材，如：追妻火葬场、闪婚、重生复仇、年代文、赘婿逆袭、规则怪谈等",
  "custom_topic": "(可选) 用户提供的具体故事核或题材",
  "reference_path": "./docs/参考资料.md",
  "config": {
    "target_chapters": 20,
    "chapter_word_min": 800,
    "chapter_word_max": 1000,
    "target_total_words": null
  }
}
```

### 【核心】选题逻辑 - 必须严格遵守

**【重要】选题决策优先级：**

```
如果用户提供了 custom_topic：
    → 直接使用 custom_topic 作为创作核心，生成完整故事

如果用户只提供了 specific_genre：
    → 在该题材类型内创作全新故事（不是照搬模板）

如果用户只提供了 user_preference：
    → 从对应类别的爆款题材池中随机选择创作全新主题
    → 【重要】必须使用创新元素，不能照搬已有模板
```

**【关键】输出路径规则：**
- 所有临时文件必须输出到 `outputs/[书名]/work/` 目录下
- **【禁止】** 禁止将临时文件输出到 `outputs/` 根目录（即使是临时文件也不行）
- 输出文件名格式：`temp_topic.json`
- 必须在调用时传入正确的 `work_dir` 参数（格式：`./outputs/[书名]/work/`）

### 输入参数说明



| 参数 | 类型 | 必填 | 说明 | 默认值 |
|------|------|--------|------|--------|
| user_preference | string | 否 | 目标读者群体 female/male/all | all |
| specific_genre | string | 否 | 具体题材：追妻火葬场/闪婚/重生复仇等 | 可选 |
| custom_topic | string | 否 | 用户提供的自定义故事核 | 可选 |
| reference_path | string | 否 | 参考资料路径 | "./docs/参考资料.md" |
| config.target_chapters | number | 否 | 目标章节数 | 20 |
| config.chapter_word_min | number | 否 | 每章最少字数 | 800 |
| config.chapter_word_max | number | 否 | 每章最多字数 | 1000 |
| config.target_total_words | number | 否 | 目标总字数（可选） | null |

### 1. 读取参考资料
- 读取 `./docs/参考资料.md` 获取爆款创作规律

### 2. 确定题材选择
**优先级判断**：
1. 如果用户提供了 `specific_genre`，直接使用该题材
2. 如果用户只提供了 `user_preference`，从对应类别的爆款题材中随机选择（非top1）
3. 如果用户未提供任何偏好，从全频爆款题材中随机选择

- 年代文

- 寡妇/弃妇逆袭
- 育儿/萌宝反套路
- 民国乱世佳人

**男频可选题材池**（排除top1都市逆袭）：

- 末世/求生
- 都市高武/全民觉醒
- 民俗/风水/相师/走阴
- 职业/特殊工种
- 历史/小人物逆袭

**全频可选题材池**：
- 阴间/地府脑洞
- 家庭/邻里矛盾

- 脑洞/万物通灵/成精

### 4. 创作新题材
**根据参考资料中的规律，创作全新主题**：

**书名公式提取**（从参考资料）：
- 女频公式：极端场景 + 背叛 + 反转
- 男频公式：身份 + 反转 + 打脸
- 脑洞公式：奇葩设定 + 冲突

**女频新主题创作方向**：
1. **追妻火葬场变体**：
   - 场景变化：不只是ICU/离婚，可以是绝症/车祸/葬礼/产房
   - 身份变化：不只是妻子，可以是秘书/设计师/医生
   - 反转时机：不只是重生，可以是穿书/觉醒/记忆恢复
   - 例：`产房外，他陪初恋笑时，我穿成了总裁千金`

2. **闪婚变体**：
   - 隐藏身份变化：总裁/军人/首富/大佬/影帝/科学家
   - 结婚原因变化：债务/赌约/家族联姻/意外怀孕
   - 身份差距变化：穷困家庭vs豪门、普通vs特殊
   - 例：`赌债百万，我嫁给了千亿战神`

3. **重生复仇变体**：
   - 穿越对象变化：继母/姐妹/恶毒女配/前男友/闺蜜
   - 穿越方式变化：知道结局/改写命运/抢先行动
   - 身份设定变化：真假千金/学霸/大佬/总裁
   - 例：`穿成假千金后，我先抢了高考状元`

4. **年代文变体**：
   - 年代变化：不只是七八十，可以是民国/古代/未来
   - 发财方式变化：经商/做小生意/摆摊/养殖/手艺
   - 家庭矛盾变化：重男轻女/吸血兄弟姐妹/极品公婆
   - 例：`八零知青：我靠倒腾买卖发家致富`

**男频新主题创作方向**：
1. **赘婿逆袭变体**：
   - 隐藏身份变化：不只是首富，可以是战神/大佬/继承权/隐世
   - 压迫方式变化：岳母/老婆/亲戚/同事/情敌
   - 摊牌时机变化：关键时刻/被迫暴露/主动摊牌
   - 例：`上门赘：三年隐忍，一朝封神`

2. **都市逆袭变体**：
   - 贫穷方式变化：不只是赘婿，可以是废物/穷小子/被赶出门
   - 身份反转：乞丐vs大佬/学生vs总裁/保安vs战神
   - 打脸场景：同学聚会/公司会议/家族宴会/慈善晚会
   - 例：`被赶出家后，我继承万亿家产`

3. **规则怪谈变体**：
   - 规则类型变化：生存规则/游戏规则/诅咒规则/灵异规则
   - 违规方式：逻辑推理/观察细节/利用漏洞/角色能力
   - 爽点密集：每关都有生死悬念
   - 例：`深夜不回家，必死：规则怪谈实录`

4. **末世求生变体**：
   - 末世类型变化：丧尸/变异/极端天气/资源枯竭/人类社会崩坏
   - 金手指变化：系统/空间/异能/物品/特殊体质
   - 团队模式：独行/小队/大团队/信任背叛
   - 例：`末世第100天，我觉醒了空间系统`

### 3. 全新主题创作方向**
1. **阴间脑洞变体**：
   - 阴间设定：地府/黄泉/奈何桥/投胎/轮回系统
   - 职业变化：摆渡/卖解忧/做红娘/当鬼差/开花店
   - 爽点密集：每个客户都有奇特故事
   - 例：`地府摆渡：我靠卖孟婆汤发家致富`

2. **家庭矛盾变体**：
   - 矛盾对象变化：极品婆婆/吸血亲戚/重男轻女父母/奇葩兄弟姐妹
   - 解决方式变化：分家/断交/法律手段/经济制裁/舆论压制
   - 爽点密集：每次冲突都要打脸成功
   - 例：`极品婆婆上门，我直接报警`

3. **校园青春变体**：
   - 身份设定变化：学霸/校霸/普通人觉醒/重生/穿越
   - 冲突类型变化：师生/同学/校规/家庭背景/霸凌弱
   - 爽点密集：考试、比赛、表白、毕业都要打脸
   - 例：`校霸借钱不还，我直接公开处刑`

4. **现实题材变体**：
   - 现实问题：房贷/加班/催婚/婆媳矛盾/职场PUA
   - 解决方式：创业/副业/投资/系统/意外机遇
   - 爽点密集：每个困难都要打脸
   - 例：`被PUA后，我裸辞开公司`

### 4. 书名生成（基于公式+创新）
- 根据选定的创作方向，应用书名公式
- 替换场景、身份、反转等关键元素
- 确保书名新颖、符合爆款规律

### 5. 核心冲突三要素（基于规律创作）
- 前期压迫：根据新主题设计具体的压迫情境
- 中期反转：根据新主题设计觉醒/摊牌/金手指方式
- 后期爽点：根据新主题设计打脸场景和结局

### 6. 预估字数和章节数

根据配置参数计算：

**计算逻辑**：
1. 如果指定了 `target_total_words`：
   - 平均章节数字数 = (chapter_word_min + chapter_word_max) / 2
   - 计算章节数 = ceil(target_total_words / 平均章节数字数)
   - 更新章节数为计算结果

2. 如果只指定了 `target_chapters`：
   - 估算总字数 = target_chapters × 平均章节数字数

3. 默认配置（20章，800-1000字/章）：
   - 总字数：1.2万-1.8万字

**字数和章节数示例**：

| 场景 | 章节数 | 每章字数 | 总字数 |
|------|--------|----------|--------|
| 短视频剧本 | 5-10 | 200-300 | 1000-3000 |
| 短篇爆款 | 20 | 800-1000 | 1.2万-1.8万 |
| 中篇小说 | 30-50 | 1000-1500 | 3万-7.5万 |
| 长篇小说 | 100+ | 1500-3000 | 15万+ |

## 注意事项

1. **书名必须符合爆款公式，提升点击率**
2. **核心冲突三要素必须完整配置**
3. **预估字数控制在1.2万-1.8万字**
4. **输出 JSON 格式供后续步骤解析**
5. **【重要】每次生成都要使用全新的角色名称**：
   - 绝不能重复使用之前的固定角色名（林婉、陆沉、苏月、陈风等）
   - 必须从提供的角色名称池中随机选择
   - 每部小说的角色名组合必须唯一
6. **【重要】题材选择要多样化**：
   - 如果用户指定了 `specific_genre`，使用用户指定的题材
   - 如果用户未指定，从对应类别的题材池中随机选择（不要总选top1）
   - 避免连续生成相同题材的小说
