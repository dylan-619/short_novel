# 番茄短篇生成系统 - 操作指南

## 一、快速上手

### 5分钟快速入门

1. **启动小说生成**
   ```
   /novel-generator-main
   ```

2. **选择偏好**
   - 女频 (female): 追妻火葬场、闪婚、重生复仇
   - 男频 (male): 都市逆袭、规则怪谈、末世求生
   - 全频 (all): 阴间脑洞、家庭矛盾、规则怪谈

3. **等待生成完成**
   - 预计时间: 30-50分钟
   - 总进度实时显示
   - 各步骤状态可见

4. **查看输出文件**
   - 小说文件: `../outputs/[书名].md`
   - 元数据文件: `../outputs/[书名]_meta.json`

### 使用主入口技能生成小说

**最简单的方式**:
```
/novel-generator-main
```

系统会自动:
- 询问目标读者偏好
- 选择最优题材
- 设计完整大纲
- 生成20章内容
- 输出完整小说

**带参数的方式**:
```json
{
  "user_preference": "female",
  "custom_topic": "重生复仇",
  "output_dir": "../outputs"
}
```

### 查看输出文件

**查看小说全文**:
```bash
cat ../outputs/[书名].md
```

**查看质量报告**:
```bash
cat ../outputs/[书名]_meta.json | jq .
```

**检查字数**:
```bash
wc -m ../outputs/[书名].md
```

### 回溯机制说明

**大纲审核不通过**
- 触发条件: 编辑审核返回 `reject` 或 `needs_revision`
- 回溯路径: 返回步骤2 (大纲师)
- 处理方式: 重新设计20章大纲

**章节审核不通过**
- 触发条件: 编辑审核返回 `needs_revision`
- 回溯路径: 重新生成该章节
- 重试次数: 最多3次
- 降级处理: 3次失败后标记为警告继续

**字数不达标**
- 触发条件: 字数不在800-1000字范围
- 处理方式: 调用字数控制技能调整
- 调整次数: 最多10次

## 三、技能详细说明

### 1. novel-step-01-topic-selector (选题师)

**职责**: 从参考资料选择最优题材，确定书名和核心爽点

**输入**:
- 用户偏好 (女频/男频/全频)
- 参考资料路径

**输出**:
- 题材分类
- 爆款书名
- 核心爽点
- 人物关系
- 基调

**执行时间**: 1-2分钟

---

### 2. novel-step-02-outline-designer (大纲师)

**职责**: 设计20章完整大纲，分配核心冲突三要素

**输入**:
- 选题师的输出

**输出**:
- 20章完整大纲
- 人物弧光
- 时间线分配

**执行时间**: 3-5分钟

---

### 3. novel-step-03-editor-reviewer (编辑审核)

**职责**: 审核大纲逻辑性、章节质量

**输入**:
- 大纲内容 / 章节内容

**输出**:
- 审核状态 (pass/reject/needs_revision)
- 质量评分
- 问题列表
- 修改建议

**执行时间**: 1-2分钟

---

### 4. novel-step-04-chapter-writer (章节写手)

**职责**: 逐章执行大纲，协调子技能完成单章写作

**输入**:
- 章节编号
- 大纲信息
- 前文内容

**输出**:
- 章节内容
- 字数统计
- 钩子信息

**执行时间**: 每章1-2分钟

**内部调用顺序**:
1. novel-sub-skill-novelist-author → 作家写作初稿
2. novel-sub-skill-word-count-controller → 控制字数
3. novel-sub-skill-hook-crafter → 设计章末钩子
4. novel-step-03-editor-reviewer → 质检
5. novel-sub-skill-reader-feedback → 读者反馈

---

### 5. novel-step-05-plot-optimizer (情节优化师)

**职责**: 优化剧情节奏，强化冲突密度

**输入**:
- 完整章节草稿
- 读者反馈

**输出**:
- 优化后全文
- 优化摘要

**执行时间**: 3-5分钟

---

### 6. novel-step-06-tone-consistency (基调维持)

**职责**: 确保全文风格统一

**输入**:
- 优化后全文
- 目标基调

**输出**:
- 基调一致性报告
- 风格偏差列表

**执行时间**: 2-3分钟

---

### 7. novel-step-07-conflict-checker (冲突检查)

**职责**: 确保核心冲突贯穿全文

**输入**:
- 基调统一后全文
- 核心冲突定义

**输出**:
- 冲突完整性报告
- 缺失冲突列表

**执行时间**: 2-3分钟

---

### 8. novel-step-08-final-compiler (最终汇编)

**职责**: 汇总所有章节，生成完整小说文件

**输入**:
- 所有章节
- 大纲信息
- 质量报告

**输出**:
- 完整小说文件
- 元数据文件

**执行时间**: 1-2分钟

---

### 9. novel-sub-skill-skill-novelist-author (作家)

**职责**: 具体章节内容写作

**输入**:
- 章节大纲
- 人物弧光
- 前文内容

**输出**:
- 章节初稿
- 字数统计

**执行时间**: 每章30秒-1分钟

---

### 10. novel-sub-skill-hook-crafter (钩子师)

**职责**: 设计章节结尾钩子

**输入**:
- 章节内容
- 下一章大纲

**输出**:
- 钩子文本
- 钩子类型
- 完整章节内容

**执行时间**: 每章10-20秒

---

### 11. novel-sub-skill-word-count-controller (字数控制)

**职责**: 精确控制每章字数

**输入**:
- 章节内容
- 目标字数范围

**输出**:
- 调整后内容
- 实际字数

**执行时间**: 每章5-10秒

---

### 12. novel-sub-skill-reader-feedback (读者反馈)

**职责**: 模拟番茄平台读者视角评价

**输入**:
- 章节内容 / 全文内容
- 目标读者群体

**输出**:
- 阅读体验评分
- 亮点和不足
- 改进建议

**执行时间**: 每章10-20秒

---

### 13. novel-generator-main (小说生成主入口)

**职责**: 协调整个小说生成流程

**输入**:
- 用户偏好
- 自定义题材（可选）
- 输出目录

**输出**:
- 完整小说文件
- 元数据文件
- 生成日志

**执行时间**: 30-50分钟

### 技能间依赖关系

```
novel-generator-main (主入口)
    ├→ novel-step-01-topic-selector
    │    ↓
    ├→ novel-step-02-outline-designer
    │    ↓
    ├→ novel-step-03-editor-reviewer
    │    ↓
    ├→ novel-step-04-chapter-writer
    │    ├→ novel-sub-skill-novelist-author
    │    ├→ novel-sub-skill-word-count-controller
    │    ├→ novel-sub-skill-hook-crafter
    │    ├→ novel-step-03-editor-reviewer
    │    └→ novel-sub-skill-reader-feedback
    │    ↓
    ├→ novel-step-05-plot-optimizer
    │    ↓
    ├→ novel-step-06-tone-consistency
    │    ↓
    ├→ novel-step-07-conflict-checker
    │    ↓
    └→ novel-step-08-final-compiler
```

### 关键参数说明

#### 选题师参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|--------|------|
| user_preference | string | 否 | female/male/all |
| reference_path | string | 否 | 参考资料路径 |

#### 大纲师参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|--------|------|
| category | string | 是 | 题材分类 |
| novel_title | string | 是 | 书名 |
| core_climax_points | array | 是 | 核心爽点 |
| relationships | object | 是 | 人物关系 |

#### 章节写手参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|--------|------|
| chapter_number | number | 是 | 章节编号 |
| outline | object | 是 | 大纲信息 |
| previous_chapters | array | 是 | 前文内容 |

#### 主入口参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|--------|------|
| user_preference | string | 否 | female/male/all |
| custom_topic | string | 是 | 自定义题材 |
| output_dir | string | 否 | 输出目录 |

### E. 常见问题FAQ

**Q: 生成一本小说需要多长时间？**
A: 预计30-50分钟，具体取决于AI响应速度和网络情况。

**Q: 可以同时生成多本小说吗？**
A: 可以，使用批量生成脚本或在不同终端运行。

**Q: 生成的小说可以直接发布吗？**
A: 质量评分9.0+的可直接发布，建议人工审核后发布。

**Q: 如何自定义题材？**
A: 使用 `custom_topic` 参数指定题材或故事核。

**Q: 可以生成不同字数的小说吗？**
A: 目前固定生成1.2万-1.8万字（20章），可修改技能调整。

**Q: 如何提高生成质量？**
A: 1) 使用高质量参考资料 2) 提供详细的自定义参数 3) 人工审核优化

**Q: 生成失败怎么办？**
A: 1) 查看错误日志 2) 检查输入参数 3) 参考故障排查章节

**Q: 可以修改已生成的小说吗？**
A: 可以，手动编辑输出文件或重新生成特定章节。

**Q: 如何导出为其他格式？**
A: 使用 pandoc 工具转换为 PDF、EPUB、TXT 等格式。
