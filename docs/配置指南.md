# 小说生成配置指南

## 概述

小说生成系统现在支持灵活的章节数和字数配置，可以适应不同的创作场景和平台需求。

## 配置参数

### 基本配置结构

```json
{
  "user_preference": "female | male | all",
  "specific_genre": "(可选) 具体题材",
  "custom_topic": "(可选) 自定义题材或故事核",
  "output_dir": "../outputs",
  "config": {
    "target_chapters": 20,
    "chapter_word_min": 800,
    "chapter_word_max": 1000,
    "target_total_words": null
  }
}
```

### 配置参数说明

| 参数 | 类型 | 忌填 | 说明 | 默认值 |
|------|------|--------|------|--------|
| target_chapters | number | 否 | 目标章节数 | 20 |
| chapter_word_min | number | 否 | 每章最少字数 | 800 |
| chapter_word_max | number | 否 | 每章最多字数 | 1000 |
| target_total_words | number | 是 | 目标总字数（可选） | null |

**计算逻辑**：
- 如果指定了 `target_total_words`，系统会自动计算合适的章节数
- 章节数 = ceil(目标总字数 / 平均章节数字数)
- 如果 `target_chapters` 和 `target_total_words` 同时指定，以 `target_chapters` 优先

## 预设配置场景

### 1. 短视频剧本（抖音/快手）

```json
{
  "user_preference": "all",
  "custom_topic": "搞笑反转",
  "output_dir": "../outputs",
  "config": {
    "target_chapters": 5,
    "chapter_word_min": 200,
    "chapter_word_max": 300
  }
}
```

**输出规格**：
- 章节数：5章
- 每章字数：200-300字
- 总字数：1000-1500字
- 特点：节奏极快，每章一个强反转

### 6. 根据总字数自动计算

适合场景：只需要控制总字数，让系统自动分配章节

```json
{
  "user_preference": "female",
  "specific_genre": "重生复仇",
  "output_dir": "../outputs",
  "config": {
    "target_total_words": 5000,
    "chapter_word_min": 400,
    "chapter_word_max": 600
  }
}
```

**计算过程**：
- 平均章节数字数 = (400 + 600) / 2 = 500
- 计算章节数 = ceil(5000 / 500) = 10章
- 输出总字数：约5000字

**输出规格**：
- 章节数：10章（自动计算）
- 每章字数：400-600字
- 总字数：约5000字

## 阶段分配规则

系统会根据总章节数自动动态分配各阶段：

| 总章节数 | 开篇冲突 | 小爽点 | 升级打脸 | 高潮决战 | 完美结局 |
|----------|----------|--------|----------|----------|----------|
| 5-10章 | 第1章 | 第2-3章 | 第4-(n-2)章 | 第(n-1)章 | 第n章 |
| 11-20章 | 第1章 | 第2-5章 | 第6-(n-3)章 | 第(n-2)-(n-1)章 | 第n章 |
| 21-50章 | 第1-2章 | 第3-8章 | 第9-(n-3)章 | 第(n-2)-(n-1)章 | 第n章 |
| 51+章 | 第1-3章 | 第4-10章 | 第11-(n-4)章 | 第(n-3)-(n-1)章 | 第n章 |

### 示例：5章分配

```
第1章：开篇冲突（300字内出冲突）
第2-3章：小爽点不断（初步打脸）
第4章：升级打脸（剧情反转）
第5章：完美结局（爽点拉满）
```

### 示例：10章分配

```
第1章：开篇冲突（300字内出冲突）
第2-3章：小爽点不断（初步打脸）
第4-8章：升级打脸（剧情反转、感情升温）
第9章：高潮决战（终极打脸）
第10章：完美结局（爽点拉满）
```

### 示例：20章分配

```
第1章：开篇冲突（300字内出冲突）
第2-5章：小爽点不断（初步打脸）
第6-17章：升级打脸（剧情反转、感情升温）
第18-19章：高潮决战（终极打脸）
第20章：完美结局（爽点拉满）
```

## 使用方法

### 方法1：直接调用主技能

```
/novel-generator-main
```

然后按照提示输入配置参数。

### 方法2：通过JSON配置

创建配置文件 `config.json`：

```json
{
  "user_preference": "female",
  "specific_genre": "闪婚",
  "output_dir": "../outputs",
  "config": {
    "target_chapters": 20,
    "chapter_word_min": 800,
    "chapter_word_max":": 1000
  }
}
```

然后调用技能：
```
/novel-generator-main config.json
```

## 注意事项

1. **字数范围合理性**：
   - 短视频：200-300字/章
   - 超短篇：400-600字/章
   - 短篇：800-1000字/章
   - 中篇：1000-1500字/章
   - 长篇：1500-3000字/章

2. **章节数合理性**：
   - 短视频：5-10章
   - 超短篇：10-15章
   - 短篇：15-20章
   - 中篇：20-50章
   - 长篇：50+章

3. **开篇冲突**：第1章300字内必须出冲突

4. **章末钩子**：每章结尾必须有强钩子（除最后一章）

5. **核心冲突三要素**：必须贯穿全文

6. **配置优先级**：
   - `target_chapters` 和 `target_total_words` 同时指定时，以 `target_chapters` 优先
   - `target_total_words` 会自动计算章节数

## 常见问题

### Q1: 如何生成抖音短视频剧本？

A: 使用短视频配置：
```json
{
  "config": {
    "target_chapters": 5,
    "chapter_word_min": 200,
    "chapter_word_max": 300
  }
}
```

### Q2: 如何控制总字数？

A: 使用 `target_total_words` 参数，系统会自动计算章节数：
```json
{
  "config": {
    "target_total_words": 10000,
    "chapter_word_min": 800,
    "chapter_word_max": 1000
  }
}
```

### Q3: 如何生成中篇小说？

A: 使用中篇配置：
```json
{
  "config": {
    "target_chapters": 30,
    "chapter_word_min": 1000,
    "chapter_word_max": 1500
  }
}
```

### Q4: 章节字数范围是否可以自定义？

A: 可以，根据你的需求设置 `chapter_word_min` 和 `chapter_word_max`，只要 `max > min` 即可。

## 技能适配说明

以下技能已经适配灵活配置：

1. ✅ novel-generator-main - 主入口，支持配置参数
2. ✅ novel-step-01-topic-selector - 选题师，支持字数配置
3. ✅ novel-step-02-outline-designer - 大纲师，支持可变章节数
4. ✅ novel-step-04-chapter-writer - 章节写手，支持可变章节数
5. ✅ novel-sub-skill-word-count-controller - 字数控制，支持动态范围
6. ✅ novel-step-03-editor-reviewer - 编辑审核，支持动态字数检查
7. ✅ novel-step-08-final-compiler - 最终汇编，支持可变章节数

其他子技能（novelist-author、hook-crafter、reader-feedback、plot-optimizer、tone-consistency、conflict-checker）无需修改，因为它们不直接依赖章节数或字数配置。
