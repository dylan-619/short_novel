# 番茄小说 2 万字短篇自动生成系统

基于 Claude Code + GLM4.7 的多 Skill 协作系统，自动生成符合番茄平台高热标准的 2 万字短篇小说。

## 项目特点

- 🚀 **一键生成**: 使用 `/novel-generator-main` 自动完成 20 章小说生成
- 📊 **质量保证**: 12 个协作 Skills 确保符合番茄平台爆款标准
- 📚 **爆款模板**: 内置 20+ 验证过的爆款题材模板
- ⚡ **高效流程**: 预计 30-50 分钟完成 1.2-1.8 万字小说
- 🔧 **灵活定制**: 支持女频/男频/全频题材选择和自定义参数

## 快速开始

### 1. 启动小说生成

```bash
/novel-generator-main
```

系统会自动：
- 询问目标读者偏好（女频/男频/全频）
- 选择最优题材并生成爆款书名
- 设计完整 20 章大纲
- 逐章生成内容
- 输出完整小说和质量报告

### 2. 查看输出文件

```bash
# 查看小说全文
cat outputs/[书名].md

# 查看质量报告
cat outputs/[书名]_meta.json
```

## 技术架构

### 12 个协作 Skills

```
┌─────────────────────────────────────────────────────────────────┐
│                    12 个协作 Skills                          │
├──────────┬──────────────────────────────────────────────────────┤
│   类别   │                   Skills 列表                     │
├──────────┼──────────────────────────────────────────────────────┤
│角色协作类│ novelist-author, editor-reviewer, reader-feedback  │
│写作流程类│ topic-selector, outline-designer, chapter-writer,  │
│          │ hook-crafter, plot-optimizer                      │
│质量控制类│ tone-consistency, word-count-controller,            │
│          │ conflict-checker                                   │
│最终整理类│ final-compiler                                      │
└──────────┴──────────────────────────────────────────────────────┘
```

### 执行流程

```
用户启动 → topic-selector (选题师)
         → outline-designer (大纲师)
         → editor-reviewer (编辑审核)
         → chapter-writer × 20 (章节写手)
              ├→ novelist-author (作家)
              ├→ hook-crafter (钩子师)
              ├→ word-count-controller (字数控制)
              ├→ editor-reviewer (编辑审核)
              └→ reader-feedback (读者反馈)
         → plot-optimizer (情节优化师)
         → tone-consistency (基调维持)
         → conflict-checker (冲突检查)
         → final-compiler (最终汇编)
         → 输出完整小说
```

## 质量标准

| 指标 | 目标值 | 检测方式 |
|------|--------|----------|
| 总字数 | 1.2-1.8 万字 | word-count-controller |
| 单章字数 | 800-1000 字 | word-count-controller |
| 开篇吸引力 | ≥ 8/10 | reader-feedback |
| 钩子覆盖率 | 100% | editor-reviewer |
| 爽点密度 | ≥ 1.5 个/章 | reader-feedback |
| 基调一致性 | ≥ 8/10 | tone-consistency |
| 冲突贯穿性 | 100% | conflict-checker |
| 读者评分预期 | ≥ 7.5/10 | reader-feedback |

## 核心设计原则

### 核心冲突三要素

| 阶段 | 要素 |
|------|------|
| 前期压迫 | 主角被伤害/看不起/陷入绝境 |
| 中期反转 | 觉醒/重生/摊牌/获得金手指 |
| 后期爽点 | 极速打脸、复仇成功、事业爱情双收 |

### 节奏铁律（20 章模板）

| 章节范围 | 内容要求 |
|----------|----------|
| 第 1 章 | 300 字内出冲突/背叛/死亡/觉醒 |
| 第 2-5 章 | 小爽点不断，初步打脸 |
| 第 6-15 章 | 升级打脸，剧情反转，感情升温 |
| 第 16-19 章 | 高潮决战，终极打脸 |
| 第 20 章 | 完美结局，爽点拉满 |

### 雷区规避（绝对不能碰）

| 雷区 | 后果 | 规避策略 |
|------|------|----------|
| 长铺垫 | 读者流失 | 开篇 300 字内必须出冲突/金手指/反转 |
| 多角恋狗血 | 节奏拖沓 | 番茄短篇读者讨厌拖沓感情戏 |
| 圣母人设 | 爽感不足 | 主角必须杀伐果断/清醒独立 |
| 水文/废话 | 完读率下降 | 每章必须有爽点/钩子/反转 |
| 烂尾 | 差评率上升 | 结局必须逻辑闭环、爽点拉满 |
| 慢热 | 粘性不足 | 2 万字内必须全程紧凑、无废话 |
